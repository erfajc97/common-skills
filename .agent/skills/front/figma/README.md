# Figma — Fuente visual del proyecto

Estas referencias son la **fuente de verdad visual** (UI/UX) del proyecto.
Se deben usar para implementar o ajustar:
- layout y grid
- espaciados (padding/margins)
- tipografías (tamaños/weights)
- componentes (tablas, botones, modales, paginación)
- colores, bordes, sombras, radios
- estados (hover/active/disabled/error/loading)
- responsive (desktop/mobile)

## Regla de trabajo (obligatoria)
1) Antes de tocar UI: revisar Figma (o capturas locales si aplica).
2) Si falta una vista/estado o no está claro en Figma: **preguntar al usuario** (no inventar).
3) Reutilizar componentes existentes. Si hace falta crear wrappers reusables (Hero UI), hacerlo solo si no existe uno equivalente.
